2) Добавить туда приложение с llm интерфейсом

Поскольку мы строим ханипот (honeypot), наше желание поместить несколько приложений (основной сервис IEC61850 + Beelzebub) в один контейнер абсолютно оправдано. Ханипоту нужно, чтобы оба сервиса (например, фейковый SSH от Beelzebub и IEC61850 сервер) «висели» на одном IP-адресе (например, 10.0.0.2), чтобы для атакующего это выглядело как одно устройство.

Самый элегантный способ объединить их, не переписывая кучу кода, — это использовать Multi-stage build (взять файлы из готового образа Beelzebub) и паттерн Entrypoint-скрипта.

 Используем универсальный стартовый скрипт (entrypoint.sh)

 Файл entrypoint.sh лежит в папке ./iec61850_vIED_honeypot/ (данный скрипт уже есть в репозитории)

Этот скрипт запустит Beelzebub в фоновом режиме, а затем передаст управление основной команде контейнера.
