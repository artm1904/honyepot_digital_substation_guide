### Шаг 1: Настройка сети (Bridge)

Это самый важный этап. При использовании Bridge ваш физический сетевой интерфейс (например, eth0) становится "портом" виртуального свитча (br0). IP-адрес хоста "переезжает" с физической карты на интерфейс моста.

В Arch Linux сетью могут управлять разные инструменты, ниже приведен пример для NetworkManager (самый популярный на десктопах). 

Сначала узнайте имя вашего проводного интерфейса:

```bash
ip link
```

Допустим, он называется enp3s0.
Вариант А: Если вы используете NetworkManager (nmcli) — Рекомендуемый

Это самый простой способ, если у вас стоит GNOME/KDE или просто установлен NetworkManager.

- Создайте интерфейс моста (br0):
    ```bash
    nmcli con add type bridge ifname br0 con-name br0
    ```

- Отключите STP (Spanning Tree Protocol), чтобы мост поднимался мгновенно (для дома STP обычно не нужен):
    ```bash
    nmcli con modify br0 bridge.stp no
    ```

- Настройте IP для моста (если хотите DHCP):
    ```bash
    nmcli con modify br0 ipv4.method auto
    ```

- (Если нужен статический IP, используйте ipv4.method manual ipv4.addresses ...)

- Добавьте физическую карту (enp3s0) в этот мост как slave:
    ```bash
    nmcli con add type ethernet slave-type bridge con-name bridge-slave ifname enp3s0 master br0
    ```

- Поднимите мост (это может временно разорвать сеть):
    ```bash
    nmcli con up br0
    ```

- Отключите старое проводное соединение (чтобы не конфликтовало), обычно оно называется "Wired connection 1" или похоже:
    ```bash
    nmcli con down "Wired connection 1" 
    # Или удалите его, чтобы не мешало автоподключению
    # nmcli con delete "Wired connection 1"
    ```

- Проверьте, что у br0 есть IP-адрес:
    ```bash
    ip addr show br0
    ```


### Шаг 2: Настройка Libvirt для использования моста

По умолчанию Libvirt создает сеть "default" (NAT). Нам нужно объяснить ему, что у нас есть мост br0, чтобы он красиво отображался в virt-manager, или мы можем просто указывать его вручную.

Лучший способ — создать XML-описание сети "host-bridge".

    Создайте файл host-bridge.xml:

```
[artm1904@archlinux t-pot]$ cat host-bridge.xml                 
<network>
  <name>host-bridge</name>
  <forward mode="bridge"/>
  <bridge name="br0"/>
</network>
```

Добавьте эту сеть в libvirt и запустите:

```
    virsh net-define host-bridge.xml
    virsh net-start host-bridge
    virsh net-autostart host-bridge
```




Результат

    Хост (Arch): Доступен по сети, ходит в интернет через br0.

    ВМ (T-Pot): При запуске она запросит IP у вашего домашнего роутера через br0. Она будет выглядеть для роутера как отдельное устройство. Другие компьютеры в сети смогут сканировать её или попадаться в её ловушки напрямую.






### Шаг 2: KVM Bridge и Multicast Snooping (Важно для GOOSE/SV)
Выполните эти команды, чтобы превратить br0 в "тупой" коммутатор, который пропускает всё:

```
# Отключаем отслеживание мультикаста на мосту
sudo ip link set br0 type bridge mcast_snooping 0

# Включаем неразборчивый режим на мосту и физической сетевой карте
sudo ip link set br0 promisc on
sudo ip link set eth0 promisc on  # Замените eth0 на имя вашей физической карты в Arch
```

### Шаг 3: Неразборчивый режим (Promiscuous) внутри ВМ Ubuntu
