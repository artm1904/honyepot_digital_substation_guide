# Описание и технические требования к инфраструктуре стенда

Развернутый программно-аппаратный комплекс (ПАК) представляет собой распределенную среду имитации киберугроз, построенную на базе модифицированной платформы **T-Pot** (https://github.com/telekom-security/tpotce) и кастомных решений для протоколов **IEC 61850**. Архитектура разделена на три функциональных сегмента: аналитическое ядро (HIVE), интеллектуальные сенсоры (Industrial/IT Sensors) и модуль искусственного интеллекта (LLM Node).

## 1. Аппаратные требования

На основе опытной эксплуатации были определены минимальные и рекомендуемые системные требования для стабильной работы ПАК. Текущая реализация на базе 4-ядерного процессора и 20 ГБ ОЗУ выявила критические узкие места, требующие оптимизации ресурсов.

Сводная таблица ресурсов ВМ:

| **Компонент (ВМ)** | **Роль в системе**      | **CPU (ядра)** | **RAM (ГБ)** | **Диск (ГБ)** | **Особенности нагрузки**                                              |
| ------------------ | ----------------------- | -------------- | ------------ | ------------- | --------------------------------------------------------------------- |
| **HIVE**           | Сбор данных и ELK       | 2+             | 8+           | 75+           | Высокая нагрузка на CPU и RAM при индексации логов.                   |
| **INDUSTRIAL/IT**  | Ханипоты и NSM          | 2+             | 8            | 75            | Загрузка CPU зависит от интенсивности трафика и работы Suricata.      |
| **LLM Node**       | Интеллектуальный бэкенд | 8+             | 10+          | 50+           | Требуется высокая производительность для обработки запросов (Ollama). Без GPU (от 8 Gb VRAM) ответы LLM 30-60 сек |


- у меня был ноут с такими характеристиками: CPU 4 ядра, 20 RAM, SSD 250 Gb +, на нем работали HIVE, IT и INDUSTRIAL (ресурсы были загружены под завязку, работало немного заторможенно)     
- **Примечание:** В текущей конфигурации использование 1 CPU и 8 ГБ RAM для узла HIVE приводило к 100% утилизации ресурсов, что вызывает задержки в визуализации данных. Для модуля LLM (Llama 3.1 8B) 8-ядерный CPU обеспечивает время отклика около 30 секунд, что является не допустимым значением для интерактивного взаимодействия.


## 2. Программная архитектура (HIVE)

Центральный узел управления (HIVE) функционирует под управлением **Ubuntu Server 24.04 LTS** и координирует работу всей инфраструктуры.

- **Контейнеризация:** Все сервисы развернуты через **Docker Compose**, что обеспечивает изоляцию и быструю масштабируемость.
    
- **Стек ELK (Elasticsearch, Logstash, Kibana):** Используется для долгосрочного хранения и анализа атак.
    
    - **Logstash:** Модернизирован для парсинга специфических логов промышленных ханипотов.
        
    - **Kibana:** Разработаны кастомные дашборды для мониторинга событий в сетях IEC 61850.
        
- **Визуализация атак:** Система включает **T-Pot Attack Map** для отображения инцидентов в режиме реального времени и **Spiderfoot** для автоматического OSINT-анализа IP-адресов атакующих.


## 3. Сегмент сенсоров и промышленных ханипотов

Данный сегмент имитирует реальную инфраструктуру высокоавтоматизированной подстанции.

### Сетевая архитектура

Для достижения максимальной реалистичности не используется стандартный проброс портов Docker. Вместо этого применен драйвер **macvlan**, который:

1. Присваивает каждому контейнеру уникальный IP-адрес из подсети `192.168.1.x`.
    
2. Делает ханипоты неотличимыми от физических устройств (IED, АРМ) для сканеров злоумышленника.
    
3. Позволяет запускать несколько сервисов на одном и том же порту (например, несколько эмуляторов IED на порту 102).
    

### Реализованные ханипоты

- **IEC 61850 vIED (Собственная разработка):** Эмуляторы терминалов релейной защиты (XCBR, PTOC, SMV) на базе `libiec61850`. Реализовано зеркалирование трафика через `tc` для анализа MMS-сессий.
    
- **IEC 61850 Simulator:** Бэкенд, имитирующий технологическую логику работы подстанции.
    
- **Beelzebub (LLM-интеграция):** «Умная приманка», интегрированная внутрь контейнеров IEC 61850. Через фоновые процессы она эмулирует стандартные IT-сервисы (SSH, Web) и генерирует ответы с помощью локальной нейросети Ollama, что позволяет затягивать атакующего в длительное взаимодействие.



## 4. Система анализа и мониторинга (NSM)

На каждом узле-сенсоре развернут комплекс инструментов анализа сетевой безопасности:

- **Suricata:** Выполняет сигнатурный анализ трафика на предмет известных эксплойтов.
    
- **P0f & Fatt:** Используются для пассивного определения ОС и извлечения фингерпринтов (TLS/SSH) без активного сканирования атакующего.
    
- **Logstash (Forwarder):** Все собранные данные в реальном времени транслируются на узел HIVE по защищенному каналу.
    

В текущем состоянии инфраструктура позволяет проводить глубокий анализ векторов атак на протоколы промышленной автоматизации, однако требует миграции на более мощные аппаратные средства для снижения времени отклика LLM-модуля и стабильной работы аналитического стека.
