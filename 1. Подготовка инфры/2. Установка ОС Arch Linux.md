# Установка базовой системы
- Если стоит цель воспроизвести мою инфраструктуру, то на ПК необходимо поставить какой-то образ Linux. Это может быть любой современный дистрибутив, но я выбрал Arch Linux.

- Если вы хотите тоже установить Arch Linux, то рекомендую ознакомитсья с парой видео на ютубе. Установка через сприпт archinstall не такая сложная (советы по установке ниже созданы LLM, за их работоспособность не ручаюсь)


## Этап 1: Подготовка

### 1. Загрузка образа

Перейдите на [официальный сайт](https://archlinux.org/download/) и скачайте ISO-образ через торрент или зеркало. Рекомендуется также скачать файл подписи (`.sig`) для проверки целостности.

### 2. Запись на флешку

- **В Linux/macOS:** Используйте команду `dd` (замените `sdX` на ваше устройство):
```
sudo dd bs=4M if=path/to/archlinux.iso of=/dev/sdX conv=fsync oflag=direct status=progress
```
- **В Windows:** Используйте **Rufus** (режим ISO) или **BalenaEtcher**.


## Этап 2: Начальная настройка (Live-окружение)

### 1. Загрузка и сеть

Загрузитесь с флешки (обычно через F12 или F2 при старте). После появления командной строки подключите Wi-Fi с помощью утилиты `iwctl`:


```
iwctl
# Внутри iwctl:
device list                           # Узнать имя интерфейса (например, wlan0)
station wlan0 scan                    # Поиск сетей
station wlan0 get-networks            # Список сетей
station wlan0 connect SSID            # Подключение (введите пароль)
exit
```

Проверьте связь: `ping google.com`.

### 2. Выполнить установку ОС Linux через скрипт archinstall

Пример выполнения скрипта:
```bash
pacman -S archinstall
archinstall
```
- дальше делать все как в ролике: https://www.youtube.com/watch?v=azGR2P5aco0&t=154s




### 3. Настройка установленной системы

Шаг 1: Проверка поддержки виртуализации

Перед установкой убедитесь, что в BIOS/UEFI включена виртуализация (VT-x для Intel или AMD-V для AMD).

Выполните в терминале:

```bash
LC_ALL=C lscpu | grep Virtualization
```

Если вывод пуст — включите виртуализацию в BIOS. Если видите VT-x или AMD-V, идем дальше.


Шаг 2: Установка пакетов

В Arch Linux пакеты для виртуализации были перегруппированы. Для полноценной работы нам понадобятся QEMU, Libvirt и графический интерфейс (virt-manager).
```bash

sudo pacman -Syu
sudo pacman -S qemu-full virt-manager virt-viewer dnsmasq vde2 bridge-utils openbsd-netcat libvirt iptables-nft
```

- qemu-full: Полный набор компонентов эмулятора.
- libvirt: Демон для управления виртуализацией.
- virt-manager: Удобный GUI для создания ВМ.
- bridge-utils: Утилиты для создания мостов.


Шаг 3: Запуск служб и настройка прав

    Запустите демон libvirtd:
    ```bash
    sudo systemctl enable --now libvirtd
    ```

    Добавьте своего пользователя в группу libvirt, чтобы запускать virt-manager без sudo:
    ```bash

    sudo usermod -aG libvirt $(whoami)
    ```

- Важно: После этой команды нужно выйти из системы и зайти снова (logout/login).
