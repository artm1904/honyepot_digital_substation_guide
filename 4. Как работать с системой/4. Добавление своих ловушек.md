Чтобы интегрировать вашу архитектуру (Сенсор) с центральным узлом T-Pot (Hive), нам нужно:

    Собрать ваши образы локально.

    Встроить их в docker-compose.yml T-Pot.

    Пробросить логи в общую директорию.

    Научить Logstash на Сенсоре читать эти логи и отправлять их на Hive.

    Настроить Kibana на Hive.

Вот пошаговое руководство.


Шаг 1: Подготовка и сборка образов на Сенсоре

Поскольку T-Pot по умолчанию скачивает готовые образы, нам нужно собрать ваши образы локально на Сенсоре, чтобы Docker Compose мог их использовать.

    Поместите папку с вашим проектом на Сенсор (например, в ~/substation_honeypot/).

    Перейдите в эту папку и выполните сборку (Docker сам найдет ваши Dockerfile из substation.yml):
    ```
    cd ~/substation_honeypot
    sudo docker compose -f substation.yml build
    ```
    Запомните имена собранных образов (обычно это имя_папки_имя_сервиса).

Шаг 2: Интеграция в T-Pot (docker-compose.yml)

Теперь нам нужно объединить ваш substation.yml с главным файлом T-Pot на Сенсоре.

Откройте micro ~/tpotce/docker-compose.yml и добавьте ваши сервисы и сеть.

1. Добавление сети:
В самом начале файла (в блоке networks) добавьте вашу подсеть:
```Yaml
networks:
  # ... старые сети t-pot ...
  subnetwork1:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 10.0.0.0/24
```
2. Добавление сервисов (IED, Simulator и т.д.):
В блок services добавьте ваши устройства.

Важное изменение: Чтобы Logstash мог прочитать ваши логи, мы должны пробросить папку /var/log/tpot из ваших контейнеров в общую папку данных T-Pot (/data).

Добавьте сервисы по этому шаблону (пример для одного IED, остальные аналогично):
```Yaml
ied1_xcbr:
    # Укажите имя образа, которое получилось после сборки в Шаге 1
    image: substation_honeypot-new-device:latest
    container_name: new-device
    hostname: new-device
    command: "/srv/libiec61850_server eth0 102 /cfg/new-device.cfg /cfg/new-device.ext R"
    networks:
      subnetwork1:
        ipv4_address: 10.0.0.2
    privileged: true
    restart: always
    volumes:
      # Пробрасываем конфиги (укажите абсолютный путь до вашей папки cfg)
      - /home/artm1904/substation_honeypot/cfg:/cfg:ro
      # МАГИЯ ЛОГОВ: Пробрасываем папку логов ханипота в общую папку T-Pot
      - ${TPOT_DATA_PATH}/substation/log:/var/log/tpot
```

Либо лучше воспользоватсья уже существующими конфигами для запуска контейнеров, как это описано для IED1_XCBR


Шаг 3: Настройка Logstash (Сенсор)

Ваши IED теперь пишут логи в ~/tpotce/data/substation/log/. Нам нужно заставить Logstash прочитать их и отправить на Hive.

1. Создание конфигурации парсера:
T-Pot пробрасывает все конфигурации Logstash изнутри своего образа. Чтобы добавить свой парсер без пересборки образа Logstash, мы "подсунем" ему наш конфигурационный файл.

Октройте файл  ~/custom_http_output.conf и добавьте данный блоки, по аналогии с уже существующими примерами, обработку логов вашей ловушки
```Logstash
input {
  file {
    # Указываем Logstash читать все файлы в этой папке
    path =>["/data/substation/log/*.log", "/data/substation/log/*.json"]
    type => "IEC61850"
    start_position => "end"
  }
}

filter {
  if [type] == "IEC61850" {
    # Если ваши сервера (C++) пишут логи в формате JSON (очень рекомендую!), раскомментируйте это:
    # json { source => "message" }
    
    # Добавляем стандартные поля T-Pot, чтобы Hive понял, откуда это
    mutate {
      add_field => {
        "t-pot_ip" => "192.168.1.126" # IP вашего сенсора
      }
    }
  }
}
```

Шаг 4: Права доступа 

Logstash внутри контейнера работает от пользователя с UID 2000 (tpot). Чтобы он мог прочитать ваши логи, папка должна иметь правильные права.

На Сенсоре выполните:
```Bash

sudo mkdir -p ~/tpotce/data/substation/log
sudo chown -R 2000:2000 ~/tpotce/data/substation/log
```



Шаг 5: Перезапуск и проверка на Сенсоре
```Bash

cd ~/tpotce
sudo systemctl stop tpot
sudo docker compose down -v
sudo systemctl start tpot
```
Затем посмотрите, нашел ли Logstash ваш конфиг:
```Bash

sudo docker logs -f logstash
```

Если всё хорошо, вы не увидите ошибок.
Шаг 6: Настройка Kibana (на HIVE-сервере)

Как только ваши IED сгенерируют первый лог, Logstash отправит его на Hive. Теперь нужно это визуализировать.

    Зайдите на Hive: https://<IP_HIVE>:64297 -> Kibana.

    Перейдите в Stack Management -> Data Views (или Index Patterns).

    Найдите паттерн logstash-* (или tpot-*) и нажмите Refresh field list (кнопка обновления вверху справа). Это нужно, чтобы Elasticsearch распознал новые поля, которые специфичны для ваших логов IEC 61850.

    Перейдите во вкладку Discover.

    В строке поиска напишите:
    type: "IEC61850"

    Вы увидите все логи от ваших новых добавленных ловушек.
