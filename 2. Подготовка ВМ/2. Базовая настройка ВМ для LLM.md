# Запуск Ollama в Docker  Linux 

- Это изолирует систему от лишних пакетов и позволяет легко удалять или обновлять LLM
Вот пошаговая инструкция, как запустить самую легкую модель и проверить работу.

### 1. Подготовка (если Docker еще не настроен)

убедитесь, что сервис Docker запущен. Если вы только что его установили:

```
sudo systemctl start docker
sudo systemctl enable docker
```


### 2. Запуск контейнера Ollama

Выполните эту команду, чтобы скачать образ и запустить контейнер в фоновом режиме. Мы пробросим порт 11434 и подключим том (volume), чтобы скачанные модели не исчезали после перезагрузки.

**Для работы на CPU (универсальный вариант):**



```
docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama
```


(Если у вас есть видеокарта Nvidia и настроен nvidia-container-toolkit, добавьте флаг --gpus=all).

### 3. Запуск самой простой модели

Для "самой простой и небольшой" модели лучше всего подходит **TinyLlama** (весит всего около 600 МБ) или **Llama 3.2 1B**. Они работают молниеносно даже на слабом CPU.

Выполните команду внутри контейнера:


```
docker exec -it ollama ollama run tinyllama
```

При первом запуске она скачается (это займет немного времени), после чего откроется диалог, где можно сразу писать запрос.

### 4. Способы проверки работы (без входа в контейнер)

Если вы хотите убедиться, что система работает "снаружи" (например, для подключения к плагинам или другим программам), используйте эти способы:

**Способ А: Проверка через браузер**  
Просто откройте в браузере ссылку:  
```
http://localhost:11434
```

Если вы увидите текст Ollama is running, значит сервер работает корректно.

**Способ Б: Проверка через API (curl)**  
Откройте терминал и отправьте тестовый запрос к нейросети через API. Это самый надежный способ проверить генерацию текста программно:

```
curl http://localhost:11434/api/generate -d '{
  "model": "tinyllama",
  "prompt": "Why is the sky blue?",
  "stream": false
}'
```

Если в ответ придет JSON с полем "response": "...", значит всё работает идеально.

### Полезные команды для управления

- **Список скачанных моделей:** docker exec -it ollama ollama list
    
- **Остановить контейнер:** docker stop ollama
    
- **Удалить контейнер (модели сохранятся в volume):** docker rm ollama
